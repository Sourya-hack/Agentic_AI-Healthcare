# MinorProject2 Full-Stack Web App

This project turns the uploaded notebook-converted Python file `minorproject2.py` into a full-stack application with:

- `backend/`: Flask API service wrapping the original ML and analysis logic
- `frontend/`: React + Tailwind dashboard with auto-generated forms
- `requirements.txt`: Python dependencies for the backend and ML stack
- `.env.example`: environment variables for local setup

The original uploaded file is preserved at `backend/storage/source/minorproject2.py` for traceability.

## What was extracted from the Python file

The app preserves and exposes:

- Dataset utilities: `clean_age`, `clean_age_capped`, `get_age_group`
- Core ML classes/functions: `ChestXrayDataset`, `build_densenet121`, `train_model`
- Error analysis helpers: `format_error_prompt`, `rule_based_llm_analysis`
- LLM path: `query_llm`
- Report simulation and label consistency utilities: `generate_simulated_report`, `detect_inconsistency`
- Explainability helper: `apply_gradcam`
- Notebook workflows converted into API tools:
  - source inspection
  - dataset preparation
  - sample image preview
  - DenseNet build
  - training
  - inference + misclassification analysis
  - structured error generation
  - LLM reasoning
  - error taxonomy
  - bias detection
  - label inconsistency detection
  - dataset-level error pattern analysis
  - Grad-CAM visualizations

Duplicate notebook-era functions like `generate_simulated_report` and `detect_inconsistency` were consolidated once without removing the underlying behavior. Their functionality is still exposed.

## Local run instructions

### 1. Backend setup

From `minorproject2-webapp`:

```powershell
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
copy .env.example .env
python backend\run.py
```

The backend runs on `http://localhost:5000`.

### 2. Frontend setup

Open another terminal in `minorproject2-webapp\frontend`:

```powershell
npm install
npm run dev
```

The frontend runs on `http://localhost:5173`.

## How to use the app

1. Open the frontend.
2. Start with `Prepare Dataset Metadata`.
3. Upload:
   - the NIH metadata CSV
   - optional train/test split text files
4. Run `Build DenseNet121`.
5. Run `Train Model` if you want to train inside the app.
6. Run downstream tools like inference, taxonomy, bias, inconsistency, and Grad-CAM.

## Where to place model files and data

This app does not hardcode Colab paths. Supply your own local paths through the UI:

- `image_dir`: folder containing the chest X-ray images
- `csv_file`: metadata CSV such as `Data_Entry_2017.csv`
- `train_list_file` and `test_list_file`: optional split lists
- model weights generated by training are kept in backend state during runtime

If you want to load pretrained weights manually later, add a `.pth` loader endpoint or place the weight file in `backend/storage/uploads/` and extend the registry.

## Notes about notebook-to-production adaptation

- Colab-only cells such as Google Drive mounting, Kaggle shell commands, and `!pip install` lines were intentionally removed from automatic startup and converted into explicit runtime inputs or documentation.
- Heavy dependencies are imported lazily where possible so the Flask server can still start even before the ML stack is fully installed.
- Async jobs are used for long-running actions like training.
- All generated artifacts are saved into `backend/storage/outputs/`.

## Deployment

### Backend

- Deploy with Gunicorn, Waitress, or another WSGI server.
- Set environment variables from `.env.example`.
- Make sure the deployment machine has enough RAM, disk, and GPU support if you plan to run training or local LLM inference.

#### Render (recommended quick setup)

This repo includes `render.yaml` for one-click backend deployment.

1. Push the project to GitHub.
2. In Render, choose **New +** -> **Blueprint** and select the repo.
3. Render reads `render.yaml` and creates `minorproject2-backend`.
4. After deploy, verify:
   - `https://<your-render-service>.onrender.com/api/health`
5. Use this backend URL in Netlify:
   - `VITE_API_BASE_URL=https://<your-render-service>.onrender.com/api`

The same `render.yaml` also includes an optional static frontend service (`minorproject2-frontend`) as a fallback to Netlify. Set `VITE_API_BASE_URL` in Render frontend service environment variables before deploying it.

Example with Waitress:

```powershell
pip install waitress
waitress-serve --host 0.0.0.0 --port 5000 backend.run:app
```

### Frontend

```powershell
cd frontend
npm install
npm run build
```

Deploy the generated `frontend/dist` folder to any static host and set `VITE_API_BASE_URL` to the deployed backend URL before building.

## Important runtime expectations

- `Train Model`, `Inference`, and `Grad-CAM` require the PyTorch stack and valid image/data paths.
- `Query Local LLM` and `LLM Error Reasoning` with `local_llm` require Hugging Face model access and significant compute resources.
- The original notebook used Colab GPU and Google Drive. Local execution may be slower and needs equivalent files available on your machine.
